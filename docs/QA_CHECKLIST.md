# 최종 검수 체크리스트

이 체크리스트는 Figma 에셋 반영 후 최종 링크와 zip을 전달하기 전에 사용합니다.

## 기본 실행

- [ ] `npm run serve`로 로컬 서버가 실행된다.
- [ ] `http://127.0.0.1:4173/index.html#home`에서 홈 화면이 열린다.
- [ ] `npm test`로 전체 route 렌더 검사가 통과한다.
- [ ] `npm run package`로 전달용 zip이 생성된다.
- [ ] `npm run deploy:preview` 또는 Vercel 배포 명령으로 검수 링크가 갱신된다.

## 화면 공통

- [ ] 375px 폭에서 좌우 가로 스크롤이 생기지 않는다.
- [ ] PC 화면에서는 375px 모바일 프레임이 중앙에 보인다.
- [ ] 모바일 화면에서는 앱처럼 전체 화면으로 보인다.
- [ ] 하단 물결 장식이 주요 버튼과 텍스트를 가리지 않는다.
- [ ] 하단 탭바가 홈/랭킹/상점/마이페이지에서 고정된다.
- [ ] 긴 화면은 세로 스크롤이 자연스럽다.
- [ ] 버튼 텍스트가 두 줄로 어색하게 깨지지 않는다.
- [ ] 카드 내부 텍스트가 넘치거나 겹치지 않는다.

## 핵심 화면

- [ ] `#loading` 로딩 화면이 보인다.
- [ ] `#onboarding1`, `#onboarding2`, `#onboarding3` 온보딩 흐름이 보인다.
- [ ] `#signup` 회원가입 화면의 로고와 소셜 버튼이 보인다.
- [ ] `#termsAgree` 약관 동의 화면에서 체크 상태가 동작한다.
- [ ] `#home` 홈 화면의 상단 유저 영역, 배너, 퀴즈 리스트, 하단 탭이 보인다.
- [ ] `#mission` 오늘의 미션 화면이 보인다.
- [ ] `#missionDone`에서 쿠키 굽기 CTA가 보인다.
- [ ] `#quiz`와 `#quizSelected`에서 답 선택 상태가 보인다.
- [ ] `#quizResultGood`, `#quizResultLow` 결과 화면이 보인다.
- [ ] `#rankingDaily`, `#rankingSeason` 랭킹 화면이 보인다.
- [ ] `#shop` 상점 화면과 상품 카드가 보인다.
- [ ] `#productDetail` 상품 상세 화면이 보인다.
- [ ] `#exchangeApply`, `#exchangeDone` 교환 신청/완료 흐름이 보인다.
- [ ] `#vault`, `#rewardDetail` 보관함/상품권 상세 화면이 보인다.
- [ ] `#myPage` 마이페이지 화면이 보인다.
- [ ] `#settings` 환경설정 화면이 보인다.
- [ ] `#inquiry`, `#inquiryDone` 문의 흐름이 보인다.

## 클릭 흐름

- [ ] 하단 홈 탭이 `#home`으로 이동한다.
- [ ] 하단 랭킹 탭이 `#rankingDaily`로 이동한다.
- [ ] 하단 상점 탭이 `#shop`으로 이동한다.
- [ ] 하단 마이페이지 탭이 `#myPage`로 이동한다.
- [ ] 홈의 오늘의 미션 버튼이 `#mission`으로 이동한다.
- [ ] 홈의 출석체크 버튼이 `#attendance`로 이동한다.
- [ ] 상점 상품 카드가 상품 상세로 이동한다.
- [ ] 상품 상세에서 교환 신청으로 이동한다.
- [ ] 마이페이지 설정 버튼이 환경설정으로 이동한다.
- [ ] 문의 작성 후 완료 화면으로 이동한다.

## 모달

- [ ] 친구 초대 모달이 열린다.
- [ ] 로그아웃 모달이 열린다.
- [ ] 마이페이지 수정 모달이 열린다.
- [ ] 퀴즈 작성 중단 모달이 열린다.
- [ ] 모달 닫기 버튼이 정상 동작한다.
- [ ] 모달 딤 영역과 본문 z-index가 어색하지 않다.

## Figma 에셋 반영 후

- [ ] 로고가 시안과 일치한다.
- [ ] 캐릭터 이미지가 잘리지 않는다.
- [ ] 오븐/온보딩 배경이 화면 비율에 맞는다.
- [ ] 상품 이미지가 흐리거나 찌그러지지 않는다.
- [ ] 아이콘 스타일이 화면 간 일관된다.
- [ ] 실제 컬러 토큰이 Figma와 크게 다르지 않다.
- [ ] 실제 폰트/굵기/행간이 시안과 크게 다르지 않다.

## 전달 전

- [ ] 검수 링크가 최신 배포를 가리킨다.
- [ ] 전달용 zip에 `.codex-local/`, `.vercel/`, `.omx/`, `delivery/`가 포함되지 않는다.
- [ ] `README.md`, `DEPLOYMENT.md`, `docs/`가 zip에 포함된다.
- [ ] 대표님께 검수용 링크가 최종 서비스 배포가 아님을 안내한다.
